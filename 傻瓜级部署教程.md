# 从零开始部署 LitFlow — 傻瓜级教程

> **⚠️ 重要：先看这里！你遇到的 "Invalid path" 错误，100% 是因为 Supabase URL 写错了。**
> 
> 我以前写成了 `https://xxx.supabase.co/rest/v1/`（多了 `/rest/v1/`），现在已经帮你修好了！

---

## 第一步：注册 Supabase（免费数据库）

1. 打开 https://supabase.com
2. 点击右上角 **"Start your project"**
3. 用 GitHub 登录（或邮箱注册）
4. 点击 **"New project"**
5. 填写：
   - **Name**：`litflow`
   - **Database Password**：设置一个密码（记下来）
   - **Region**：选 **Singapore**
   - **Pricing Plan**：选 **Free**
6. 点击 **"Create new project"**
7. **等待 1-2 分钟**

---

## 第二步：获取正确的密钥（这一步最容易出错）

项目创建好之后：

### 2.1 获取 Project URL 和 anon key
1. 左侧菜单点击 **Project Settings**（⚙️齿轮图标）
2. 左侧找到 **API** 并点击
3. 你会看到两个重要信息：
   - **Project URL**：长这样 👉 `https://xxxxx.supabase.co`
   - **anon public**：长这样 👉 `eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...`（一大串）

### 2.2 填写到 .env.local
打开项目里的 `.env.local` 文件，把上面两个值填进去：

```
NEXT_PUBLIC_SUPABASE_URL=https://你复制的那串URL（不要加/rest/v1/）
NEXT_PUBLIC_SUPABASE_ANON_KEY=eyJ...你复制的长串anon key
```

✅ **关键检查点**：
- URL **不能**以 `/rest/v1/` 结尾，必须是 `.co` 结尾
- anon key **必须**以 `eyJ` 开头

### 2.3 运行 SQL 建表
1. 左侧点击 **SQL Editor**
2. 点击 **"+ New Query"**
3. 用记事本打开项目里的 `supabase/schema.sql` 文件
4. **全部复制**，粘贴到网页上的编辑框
5. 点击 **"Run"**（运行）
6. 看到绿色的 "Success" 就成功了 ✅

---

## 第三步：获取 DeepSeek API Key（AI功能，可跳过）

> 跳过后除了AI生成提纲不能用，其他功能都正常

1. 打开 https://platform.deepseek.com
2. 注册并登录
3. 左侧点击 **"API Keys"**
4. 点击 **"+ Create API Key"**，取名叫 `litflow`
5. 复制生成的 Key
6. 填入 `.env.local`：

```
DEEPSEEK_API_KEY=你复制的key
```

---

## 第四步：启动项目！

1. 在 VSCode 顶部菜单：**终端 → 新建终端**
2. 输入命令：
```bash
npm run dev
```
3. 看到 `http://localhost:3000` 后，按住 **Ctrl 点击** 它
4. 浏览器中：
   - 点击 **"开始使用"**
   - 点击 **"注册"**
   - 输入邮箱和密码（密码至少6位）
   - 点击 **"注册"**
5. 去邮箱查收验证邮件，点击验证链接
6. 回到登录页登录

### 如果注册时还报错？
可能是邮箱验证的问题。去 Supabase 关掉验证：
1. Supabase 左侧点击 **Authentication → Providers**
2. 找到 **Email**，点 **Edit**
3. 关掉 **"Confirm email"** 按钮
4. 点 **Save**
5. 重新注册就能直接登录了，不用等邮件
